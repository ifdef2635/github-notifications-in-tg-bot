Основные компоненты
✅ Telegram Bot (main.py, bot_handlers.py):

Регистрация пользователей по команде /start.

Рассылка уведомлений о дедлайнах.

Ежедневная автоматическая проверка в 9:00 UTC (через APScheduler).

✅ GitHub API Integration (github_handler.py):

Получение списка открытых задач (get_issues()).

Парсинг дедлайнов из:

Milestone задачи (issue.milestone.due_on).

Текста задачи (по шаблону Дедлайн: YYYY-MM-DD).

✅ База данных SQLite (database.py):

Хранение chat_id подписчиков.

Методы для добавления, удаления и получения списка пользователей.

✅ Конфигурация (config.py):

Загрузка переменных окружения (GITHUB_TOKEN, REPO_NAME, TELEGRAM_TOKEN).

Инициализация клиента GitHub.

2.2. Логика уведомлений
Бот проверяет дедлайны и отправляет уведомления:

За 10, 5, 3, 2, 1 день до дедлайна.

При просрочке (ежедневно, пока задача не закрыта).
